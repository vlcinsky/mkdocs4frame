definition=Navigation Message Authentication. The “Authentication Function” assesses the trust status of each GNSS constellation/system for which Galileo provides the relevant authentication.
order=0
Notes: <u>Overall status of the NMA service.</u><br/>This field is an integer number coded as below. (Value – Status – Meaning)<br/><ul>
<li>0 – Reserved – not used at the moment</li><li>1 - Test - NMA is provided without any operational guarantees</li><li>2 - Operational - NMA is provided according to the specifications</li></ul>
<br/>This attributes is to use Galileo Open Service Navigation Message Authentication that is specified in “Galileo Navigation Message Authentication Specification for Signal-In-Space Testing – v1.0, GROW/J1/IHF/kp/ARES(2016)6611263).<br/>The “Authentication Function” assesses the trust status of each GNSS constellation/system for which Galileo provides the relevant authentication. It can:<br/><ul>
<li>Run in the tracking device or in an external Service Platform</li><li>Be combined with the implementation of EGNOS/EDAS/multi-GNSS based services (in this case it is combined with the “Processing Algorithm” above detailed). There are two possibilities:</li><li>Galileo Open Service authentication only. In this case, the capability of supporting EGNOS is not required (though it is commonly available);</li><li>Galileo Open Service authentication combined with the EGNOS/EDAS/multi-GNSS based services. In this case, the information/data from the “Authentication Function” are the “Optional data” of the raw data message.</li></ul>
<br/>origin: CWA 16390:2018<br/>

definition=Status of authentication for each satellite ("false" = not authenticated; "true" = authenticated).
order=1
Notes: Status of authentication for each PRN.<br/>This field is a bit mask, reporting two possible values for each supported PRN: (Value – Status – Meaning)<br/><ul>
<li>0 – not authenticated – authentication information is not available for this PRN or the authentication verification is failed</li><li>1 – authenticated - authentication information is available for this PRN and the authentication verification is successful</li></ul>
